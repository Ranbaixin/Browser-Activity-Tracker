# 核心逻辑详细解析 (/public)

本文件夹包含了插件的所有生产环境代码。以下是详细的代码逻辑说明：

## 1. `background.js` (后台脚本)

这是插件最核心的文件，运行在独立的 Service Worker 进程中。

### 核心函数解析：
*   **`getActiveTab()`**: 
    *   **逻辑**：首先查询 `lastFocusedWindow` 的活跃标签。如果发现当前聚焦的是插件自己的窗口（如独立小窗口），它会自动回退并寻找最近的一个 `windowType: 'normal'`（普通浏览器窗口）的活跃标签。
    *   **目的**：解决点击小窗口时计时停止的 Bug。
*   **`tick()`**: 
    *   **逻辑**：每秒触发一次。它会检查黑名单、闲置状态，并计算 `delta`（增量）。
    *   **广播**：在函数末尾，它会发出 `TIMER_TICK` 消息，将当前时间同步给所有打开的 Popup 或独立窗口。
*   **`updateStorage()`**:
    *   **逻辑**：采用合并算法。如果当前访问与上一条历史记录的时间间隔小于 10 秒，则直接更新旧记录的 `endTime`，而不是创建新记录，从而极大地节省了存储空间。

## 2. `content.js` (内容脚本)

负责在用户浏览的网页上渲染悬浮时钟。

### 核心逻辑：
*   **隔离性**：通过检查 `window.hasTrackerInjected` 防止重复注入。
*   **实时更新**：它不自己计算时间，而是每秒向 `background.js` 请求 `GET_CURRENT_SESSION`，确保显示的数字与后台数据库完全一致。
*   **定位功能**：监听 `LOCATE_BALL` 消息，收到后会将悬浮球强制移动到屏幕中心 `(50%, 50%)` 并触发 CSS 脉冲动画。

## 3. `popup.js` (弹出窗口逻辑)

负责 UI 交互和设置管理。

### 核心逻辑：
*   **实时同步**：监听后台发来的 `TIMER_TICK` 消息。这意味着即使您不刷新 Popup，顶部的蓝色计时器也会每秒跳动。
*   **设置联动**：当您修改颜色或尺寸时，它会触发 `SETTINGS_UPDATED` 广播，让所有已打开网页上的悬浮球立即变色变大，无需刷新网页。

## 5. ⚠️ 兼容性与限制

*   **浏览器支持**：完全兼容 **Google Chrome** 和 **Microsoft Edge**。
*   **系统页面限制**：由于浏览器安全策略，悬浮球无法在 `chrome://` 或 `edge://` 等系统页面显示。
*   **特定网站限制**：由于部分网站（如 **哔哩哔哩 Bilibili**）具有极强的全局样式保护和复杂的 DOM 结构，悬浮球可能无法在这些页面正常显示或定位。
*   **刷新生效**：在安装或更新插件后，您需要**刷新**已打开的网页，悬浮球才能正常加载。
